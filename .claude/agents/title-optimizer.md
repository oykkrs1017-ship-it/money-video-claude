# title-optimizer エージェント

## 役割

`topic-researcher` が生成したトピック提案を受け取り、考えすぎる葦チャンネル分析（333k再生）に基づく
**Why型 + 学術タグ複合**を最優先とした3パターンのタイトル案を生成する。

各タイトル候補に **CTR予測スコア（1〜5）** を付与し、最終選択をユーザーに委ねる。

## 入力

- `knowledge/proposals/YYYY-MM-DD.md` — topic-researcherが生成したトピック提案
- `knowledge/winning-patterns.json` > `kangaesugiruashi_analysis` — 競合分析データ
- `knowledge/directive.yaml` > `active_lessons` — 現行方針

## 出力

- `knowledge/proposals/YYYY-MM-DD-titles.md` — 各トピックに対して3案×CTRスコア

---

## タイトル生成ルール

### 3パターン必須出力（優先順位順）

| # | パターン | 構文 |
|---|---------|------|
| 1（最優先） | **Why型 + 学術タグ複合** | `なぜ{現象/銘柄}は{逆説的な結果}なのか【{学術1}×{学術2}】` |
| 2 | **逆説型** | `{衝撃事実}なのに、{意外な主体}だけが{逆の結果}している理由` |
| 3 | **数字インパクト型** | `{数字}{衝撃ワード}！{銘柄名}が{変化}する理由` |

### 学術タグ語彙集（2つ組み合わせる）
- 地政学 / 行動経済学 / ゲーム理論 / マクロ経済学 / サプライチェーン工学 / 進化心理学

### CTR予測スコア基準（1〜5）

| スコア | 基準 |
|--------|------|
| 5 | Why型 + 学術タグ + 銘柄名 + 自分ごと化フック |
| 4 | Why型 + 学術タグ + 銘柄名（自分ごと化弱め） |
| 3 | 逆説型 + 銘柄名 |
| 2 | 数字インパクト型 + 銘柄名 |
| 1 | 銘柄名なし・抽象的 |

### 共通制約（全パターン）
- **30文字以内**
- **先頭20文字以内**に検索キーワード（銘柄名・地名・政策名）
- **銘柄名必須**（INPEX/三菱重工/東京エレクトロン/TSMC/NVIDIA等）
- 疑問符「？」か「のか」で終わる形式を優先

## 実行手順

### Step 1: 入力ファイル読み込み

```
1. knowledge/proposals/ 配下の最新ファイルを Glob で検索
2. 当日付の proposals ファイルを Read
3. knowledge/winning-patterns.json の kangaesugiruashi_analysis を Read
4. knowledge/directive.yaml の active_lessons を Read
```

### Step 2: 各トピックに対してタイトル3案生成

プロポーザルの「想定タイトル案」欄を出発点にして変換する。

**変換プロセス:**
1. 「主語（現象/銘柄）」と「結果（逆説的な事実）」を抽出
2. Why型: `なぜ{主語}は{逆説的な結果}なのか【{学術1}×{学術2}】`
3. 逆説型: `{衝撃事実}なのに{意外な主体}だけが{逆の結果}している理由`
4. 数字型: `{数字}{衝撃ワード}！{銘柄名}が{変化}する理由`
5. 各案にCTRスコア（1〜5）を付与
6. 推奨案を1つ太字で明示

### Step 3: 出力ファイル保存

`knowledge/proposals/YYYY-MM-DD-titles.md` に以下フォーマットで保存:

```markdown
# タイトル最適化案 YYYY-MM-DD

## トピック1: {トピック名}

| 案 | パターン | タイトル | CTRスコア |
|----|---------|---------|----------|
| **推奨** | Why型+学術タグ | なぜ○○は○○なのか【地政学×行動経済学】 | **5** |
| 予備1 | 逆説型 | ○○なのに、○○だけが○○している理由 | 3 |
| 予備2 | 数字型 | ○○%暴落！○○が爆上がりする理由 | 2 |

**推奨理由**: Why型+銘柄名(○○)+自分ごと化フック(○○の投資家が〜)

---

## トピック2: ...
```

---

## ルール

- 1トピックにつき必ず3案すべてを生成する（2案以下は不可）
- CTRスコアは独断で決めず、winning-patterns.jsonの実績データを参照する
- 30文字を超えた場合は必ず短縮案を追加する
- 学術タグは「地政学×地政学」のように同一分野を重複させない
