# 学び: プロンプトのスキーマ例示が出力形式を決める

日付: 2026-05-09
カテゴリ: 上位原則候補
関連スキル: generate-ep, generate-script

## 状況

`rich-panel` の `points` フィールドを文字列配列（`["ポイント1", "ポイント2"]`）として
SYSTEM_PROMPT に例示していた。Claude はその形式通りに生成し続けていたため、
KPI数値・解説文・出典などの豊かなデータが出力されていなかった。

オブジェクト形式（`{text, value, unit, body, source}`）に例示を変えたところ、
次回からその形式で自動生成されるようになった。

## 齟齬の内容

- AI の判断: 既存の `points: ["string"]` 形式が仕様だと解釈し、それを守った
- ユーザーの意図: スライドに情報密度をもたせたい（KPI・出典・解説を含めたい）
- 差分の本質: 「どんなデータ構造が可能か」をAIに伝える手段がプロンプトの例示しかなかった

## 学び

**LLMへの出力スキーマ指示は「例示」で決まる。仕様説明だけでは不十分。**

- `points: ["テキスト"]` という例があれば、Claudeは文字列配列しか生成しない
- `points: [{text: "...", value: "88%", body: "..."}]` という例を見せると、即座にその形式を使う
- 「使えるフィールド」の説明文だけではなく、**実際に使ったYAMLの例**を必ず入れること
- 必須ルール側にも「オブジェクト形式で書くこと」と明記しないと、簡単な形式に戻りがち
